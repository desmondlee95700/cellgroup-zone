# Frontend Design Evals

These evals are seed prompts for testing whether the skill changes frontend outcomes in useful ways. They mix subjective design quality with concrete, checkable expectations.

Recommended workflow:

1. Snapshot the old skill before editing when comparing versions.
2. Run each prompt once with the updated skill and once with the baseline or previous skill.
3. Save outputs under a sibling workspace such as `frontend-design-workspace/iteration-1/`.
4. Grade the expectations in `evals.json`.
5. Generate a review page with the skill-creator viewer so a human can compare the visual outputs.

Design evals need human judgment. The expectations catch obvious misses, but screenshots and qualitative review matter more than pass rate alone.
